# HQ Platform - Единая платформа для управления агентством

## Описание

HQ Platform - это единая платформа для:
- **Agency Ops** - управления агентством (операционка)
- **Personal Finance** - личного финансового дашборда (личные деньги/цели)
- **Agency Management Analytics** - управленческой аналитики по агентству (клиенты/услуги/выручка)

## Технологии

- Next.js 16 (App Router)
- TypeScript
- Prisma ORM + SQLite
- Tailwind CSS

## Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка базы данных

База данных находится в `prisma/dev.db` (SQLite).

Создайте миграции и заполните базу тестовыми данными:

```bash
npx prisma generate
npx prisma migrate dev
npm run db:seed
```

**Примечание:** Если seed не работает из-за проблем с Prisma 7, попробуйте запустить через Node.js напрямую:
```bash
export DATABASE_URL="file:./prisma/dev.db"
node --loader tsx prisma/seed.ts
```

Или создайте базу данных вручную через Prisma Studio и заполните тестовыми данными.

### 3. Переменные окружения

Создайте файл `.env.local` в корне проекта:

```env
DATABASE_URL="file:./prisma/dev.db"
OWNER_PASSWORD="owner123"
PM_PASSWORD="pm123"
```

### 4. Запуск приложения

**Локально (рекомендуется — в системном Terminal, не в Cursor):**

```bash
cd /Users/admin/Desktop/cursor/twinworks
npm run dev
```

Открой в браузере: **http://localhost:3000**  
Дашборд: http://localhost:3000/me/dashboard

Если в Cursor терминале порт выдаёт `EPERM` — запускай те же команды в **Terminal.app** или iTerm на Mac.

**На Vercel:**

```bash
npx vercel --prod
```

После деплоя открой выданный URL (например `https://twinworks-xxx.vercel.app`).  
Если дашборд долго грузится — через 18–20 сек появится сообщение об ошибке и кнопка «Повторить».

## Вход в систему

### OWNER (владелец)
- **Пароль:** `owner123`
- **Доступ:** Все модули (`/ops`, `/me`, `/manage`)

### PM (проджект-менеджер)
- **Пароль:** `pm123`
- **Доступ:** Только модуль операционки (`/ops`)
- **Ограничение:** Не видит финансовые данные проектов

## Структура модулей

### 1. Agency Ops (`/ops`)

**Цель:** Видеть загрузку команды по дням и управлять срочными задачами.

#### Страницы:
- `/ops` - Week view (загрузка команды по дням недели)
- `/ops/projects` - Список проектов
- `/ops/projects/[id]` - Детали проекта
- `/ops/planned` - Плановые задачи
- `/ops/interrupts` - Срочные задачи (с быстрым добавлением)

#### Особенности:
- Цветовая индикация загрузки: зеленый (≤80%), желтый (80-100%), красный (>100%)
- Быстрое добавление срочных задач (кнопка "+ Срочное")
- Автоматическое распределение плановых задач по рабочим дням

### 2. Personal Finance (`/me`) - только OWNER

**Цель:** Управление личными финансами и целями.

#### Страницы:
- `/me/dashboard` - Дашборд с балансами, расходами и целями
- `/me/accounts` - Управление счетами
- `/me/transactions` - Транзакции (доходы/расходы/переводы)
- `/me/goals` - Цели месяца и квартала

#### Особенности:
- Группировка счетов: "Касса", "Подушка", "На цели"
- Отслеживание расходов месяца vs ожидания
- Прогресс-бары для целей
- Возможность привязки целей к счетам (автообновление)

### 3. Agency Management Analytics (`/manage`) - только OWNER

**Цель:** Управленческая аналитика по агентству.

#### Страницы:
- `/manage` - Обзор: выручка, топ клиентов, топ услуг
- `/manage/clients` - Детали по клиентам (пока не реализовано)
- `/manage/services` - Детали по услугам (пока не реализовано)
- `/manage/projects/[id]` - Финансовая информация по проекту (пока не реализовано)

#### Особенности:
- Фильтрация по месяцам
- Расчет выручки по клиентам, услугам, проектам
- PM не имеет доступа к этим данным

## База данных

База данных SQLite находится в `prisma/dev.db`.

### Основные сущности:

**Agency Ops:**
- `TeamMember` - сотрудники
- `Client` - клиенты
- `Project` - проекты
- `PlannedItem` - плановые задачи
- `Interrupt` - срочные задачи

**Management Analytics:**
- `Service` - услуги
- `ProjectFinancial` - финансовая информация по проектам
- `Payment` - платежи

**Personal Finance:**
- `PersonalAccount` - счета
- `PersonalTransaction` - транзакции
- `PersonalGoal` - цели
- `PersonalSettings` - настройки

## Демо-данные

После запуска seed создаются:
- 4 сотрудника (PM, Designer, Copy, Developer)
- 3 клиента
- 5 проектов
- Плановые задачи и срочные задачи на текущую неделю
- Услуги и платежи
- Личные счета, транзакции и цели
- Ожидаемые расходы: 109,000 ₽/месяц

## Безопасность

- PM не может получить доступ к `/me` и `/manage` (проверка на уровне сервера)
- Все API роуты проверяют роль пользователя
- Финансовые данные проектов не отображаются в интерфейсе для PM

```bash
npx prisma studio    # Визуальный редактор БД
npx prisma migrate dev  # Создание и применение миграций
```

## Решение проблем

### Seed не работает

Убедитесь, что:
1. DATABASE_URL указан в `.env.local`
2. База данных создана (`npx prisma migrate dev`)
3. Prisma Client сгенерирован (`npx prisma generate`)

Если проблема с импортом Prisma Client в seed.ts, попробуйте:
```bash
export DATABASE_URL="file:./prisma/dev.db"
npm run db:seed
```

### Ошибки импорта Prisma Client

Если возникают ошибки с импортом `@prisma/client`, выполните:
```bash
npx prisma generate
```
